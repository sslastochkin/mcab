    Getting Data¶

    rdata = RandomData(42)

def get_covariate(data):
    return pd.Series(data + np.random.normal(0, data.std(), size=len(data)), name='covariate')

def get_strats(data):
    return pd.Series(
        pd.qcut(
            data+np.random.normal(0, 1e-6, size=len(data)),
            q=10,
            duplicates='drop',
            labels=[f"decile_{i}" for i in range(1, 11)]
        ).astype(str),
        name='strat'
    ).to_frame()

def get_union(covariate, strats):
    return pd.concat([covariate, strats], axis=1)

# Exponential data. Example - ARPU metric
arpu_data_raw  = rdata.exponential_data(size=10_000)
arpu_covariate = get_covariate(arpu_data_raw)
arpu_strats    = get_strats(arpu_data_raw)
arpu_X         = get_union(arpu_covariate, arpu_strats)

#Ttest pvalue
pval_func = lambda test, control: stats.ttest_ind(test, control, equal_var=False).pvalue

    Reducing variance¶

    Lets start with simple iid data case.
    First thing we will see - CUPED results. 

    cuped = CUPED()

# cuped transformed data
cuped_arpu_data = arpu_data_raw - cuped.fit_predict(arpu_covariate, arpu_data_raw)

# cuped score
cuped_score = cuped.score(arpu_covariate, arpu_data_raw)

    How we can see, cuped reduced ~50% of variance.
    Now lets check out poststratification. 

    post_strat = PostStratification()

# post-stratification reduction
pstrat_arpu_data = arpu_data_raw - post_strat.fit_predict(arpu_strats, arpu_data_raw)

# post-stratification score
pstrat_score = post_strat.score(arpu_strats, arpu_data_raw)

    Post-stratification in our case dealing better.

    Cuped and post-stratification are special cases of linear regression.
    What if we will combine them?

    Let's build a new model, called cupac, and see the variance reduction results.

    cupac = CUPAC(LinearRegression(fit_intercept=True, copy_X=False))
cupac.fit(
    arpu_X,
    arpu_data_raw,
    cat_cols=['strat'],
    cat_encoder=lambda categories: OneHotEncoder(
        categories=categories,
        drop='first',
        sparse_output=False
    )
)
cupac_score = cupac.score(arpu_X, arpu_data_raw)

    It looks not much better than post-stratification.
    But anyway, why should we select the one if we can select the both?
    CUPAC class with relevant variables allow us do that.

    Running simulations¶

    Let's initiate designers and load the data.

    cuped_transform  = lambda X, y: y - CUPED().fit_predict(X, y)
pstrat_transform = lambda X, y: y - PostStratification().fit_predict(X, y)
cupac_transform  = lambda X, y: y - CUPAC('ols').fit_predict(X, y, cat_cols=['strat'])

    arpu_data       = AaDataIid(data=arpu_data_raw)
arpu_data_cupac = AaDataIid(data=arpu_data_raw, transformer=cupac_transform, covariates=arpu_X)

des_iid       = DesignerIid(arpu_data,       name='arpu',       pval_func=pval_func)
des_iid_cupac = DesignerIid(arpu_data_cupac, name='arpu_cupac', pval_func=pval_func)

    Now lets check I type error.

    aa_pvalues = des_iid.aa_sims(n_sims=10_000, n_jobs=-1, verbose=False)


    aa_pvalues_cupac = des_iid_cupac.aa_sims(n_sims=10_000, n_jobs=-1, verbose=False)

    Lets compare the power.

    designers_list = [des_iid, des_iid_cupac]
bm = BenchMarker(designers_list)
ab_stats, power_curves = bm.ab_benchmark(
    bar_effect=0.05,
    power_curve_effects=[0.01, 0.05, 0.07, 0.09, 0.1],
    verbose=False,
    n_jobs=-1
)

    Signs agreement¶

    The next thing we should think about is signs agreement.
    Lets check it out. 

    analyze_results = bm.directional_analysis(
    designer1=des_iid,
    designer2=des_iid_cupac,
    n_jobs=-1,
    n_sims=10_000,
    verbose=False,
    effect=0.05
)

    We see that methods mostly agreed, but not everytime.
    Sign disagreement could cause a problem we should care about.
    If our raw target shows difference between test and control in one side and it's variance reducted copy shows us effect in other side we can not interpret this effect correctly.

    By this reason we need have extra design for a case when signs differ.
    When time for estimate experiment by variance reducted data has come, we look at the sign of our variance reducted metric.
    If it differ from sign of raw data sign we should continue experiment and estimate results with another way.

    Ratio Data¶

    Most safety way to variance reduction with ratio data is reduct the linearized data.
    MCAB has ready implemented backend for such operations.
    It works the next way: 1. Given effect adds to test data 2. Applying linearization 3. Applying transformer (variance reduction).

    # Avg bill data
avg_bill_data_raw  = rdata.ratio_data_avg_bill(size=10_000)
avg_bill_covariate = get_covariate(avg_bill_data_raw[0]/avg_bill_data_raw[1])
avg_bill_strats    = get_strats(avg_bill_data_raw[0]/avg_bill_data_raw[1])
avg_bill_X         = get_union(avg_bill_covariate, avg_bill_strats)
#
avg_bill_data        = AaDataRatio(avg_bill_data_raw)
avg_bill_data_cuped  = AaDataRatio(avg_bill_data_raw, transformer=cuped_transform,  covariates=avg_bill_covariate)
avg_bill_data_pstrat = AaDataRatio(avg_bill_data_raw, transformer=pstrat_transform, covariates=avg_bill_strats)
avg_bill_data_cupac  = AaDataRatio(avg_bill_data_raw, transformer=cupac_transform,  covariates=avg_bill_X)
#
des_avg_bill        = DesignerRatioLin(avg_bill_data,        ratio='float', name='avg_bill',        pval_func=pval_func)
des_avg_bill_cuped  = DesignerRatioLin(avg_bill_data_cuped,  ratio='float', name='avg_bill_cuped',  pval_func=pval_func)
des_avg_bill_pstrat = DesignerRatioLin(avg_bill_data_pstrat, ratio='float', name='avg_bill_pstrat', pval_func=pval_func)
des_avg_bill_cupac  = DesignerRatioLin(avg_bill_data_cupac,  ratio='float', name='avg_bill_cupac',  pval_func=pval_func)
#
designers_avg_bill = [
    des_avg_bill,
    des_avg_bill_cuped,
    des_avg_bill_pstrat,
    des_avg_bill_cupac
]
bm_avg_bill = BenchMarker(designers_avg_bill)

    We can challenge variance reduction right from BenchMarker.
    First designer in the list will be used as baseline level. 

    compare_variance = bm_avg_bill.compare_variance()

    Also we have remember to control first type error.

    aa_pvalues, aa_stats = bm_avg_bill.aa_benchmark(n_jobs=-1, verbose=False)

    Cupac still giving the best results.

    Balancing, Weighting and Advanced techniques¶

    MCAB doesn't have ready implemented methods for balancing and weighting, but it has flexible api able to integrate this methods to MCAB backend.
    All you need a transformer function with target outcome by giving covariates.
    Flow is follows:
    - Adding effect (if needed) - Applying linearization (if needed) - Applying transformation <- your function will call here - Applying pvalue function <- you can define whatever you want with test and control returning pvalue
